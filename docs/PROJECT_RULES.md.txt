# PROJECT_RULES.md

# Reglas Oficiales de Desarrollo

Proyecto:

OSINET Time ERP

Versión:

1.0

---

# Objetivo

Este documento establece las reglas oficiales para el desarrollo de OSINET Time ERP.

Todo nuevo módulo deberá cumplir estas reglas.

---

# Regla No. 1

Nunca romper funcionalidades existentes.

Cada módulo nuevo debe ampliar el sistema sin eliminar ni modificar el funcionamiento correcto de módulos anteriores.

---

# Regla No. 2

Siempre entregar archivos completos.

Nunca trabajar con fragmentos de código.

Todos los archivos deben estar listos para:

- Copiar
- Pegar
- Compilar

---

# Regla No. 3

Mantener la arquitectura del proyecto.

La estructura oficial es:

database/

repository/

model/

engine/

ui/

navigation/

utils/

docs/

---

# Regla No. 4

Patrones oficiales

El proyecto utiliza:

- MVVM
- Repository Pattern
- Engine Pattern
- StateFlow
- Navigation Compose

---

# Regla No. 5

Base de datos

Toda la información local se almacena utilizando Room Database.

No utilizar SQLite directamente.

---

# Regla No. 6

Interfaz

Toda la interfaz debe desarrollarse con:

Jetpack Compose

Material Design 3

---

# Regla No. 7

Motores (Engine)

Toda lógica de negocio compleja debe implementarse mediante motores independientes.

Ejemplos:

PayrollEngine

PayrollPdfEngine

WhatsAppPayrollEngine

WhatsAppAttendanceEngine

WhatsAppNotificationEngine

---

# Regla No. 8

Motores reutilizables

Cada Engine debe poder reutilizarse desde cualquier pantalla sin duplicar lógica.

---

# Regla No. 9

Compatibilidad

Android 9+

---

# Regla No. 10

Escalabilidad

Cada módulo debe prepararse para funcionar posteriormente con:

- API REST
- Portal Web
- Sincronización Cloud
- Inteligencia Artificial
- WhatsApp
- n8n

---

# Regla No. 11

Documentación

Cada nuevo módulo deberá actualizar:

README.md

CHANGELOG.md

PROJECT_CONTEXT.md

OSINET_PROJECT_GUIDE.md

cuando corresponda.

---

# Regla No. 12

Expediente Digital

Todo documento generado por el sistema debe poder asociarse al expediente del empleado.

Ejemplos:

- PDF Nómina
- PDF Vacaciones
- Permisos
- Amonestaciones
- Certificados
- Evaluaciones

---

# Regla No. 13

WhatsApp

Ningún empleado podrá acceder a información de otro empleado.

La validación siempre se realizará mediante el número telefónico registrado en la ficha del empleado.

---

# Regla No. 14

Seguridad

Toda información personal debe permanecer protegida.

Cada empleado solo podrá consultar:

- Su asistencia
- Su nómina
- Sus documentos
- Sus vacaciones
- Sus permisos

Nunca información de otros empleados.

---

# Regla No. 15

Calidad del código

Todo código nuevo debe mantener:

- Organización
- Claridad
- Escalabilidad
- Compatibilidad
- Reutilización

---

# Regla No. 16

Visión del proyecto

OSINET Time ERP no es únicamente un sistema de asistencia.

Es un ERP completo de Recursos Humanos para empresas de la República Dominicana.

Todas las decisiones técnicas deberán tomarse pensando en el crecimiento futuro del sistema.
